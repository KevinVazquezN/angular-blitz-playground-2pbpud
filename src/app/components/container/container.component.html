Welcome to the Blitz challenge!

We know this is not the greatest-looking site, yet the point of it is to see a bit of what you're capable of while developing in Angular. So, yes, you guessed it right, we will not necessarily judge the styles you apply to your solution.
For this exercise, we expect you to fulfill a couple of requirements. So, get ready to start having fun!
The first thing we expect you to do is to go ahead and this project to make your very own unique solution.

Now, we're sure you've already noticed this, but if not, at the top of this page there is a kind-of menu, where there are two pages available: Instructions and Users. Yes, you could go now and try going back and forth between them, just come back here to keep reading these instructions.

Great, now that you're back let's start with the requirements you're ought to make.

1

If you go to the users-list.component.html you will notice each of the columns is manually defined in the code. In Blitz, we're all in for the dynamic definition of objects, so, we would love to see how you implement these columns dynamically. Meaning that instead of having n-number of columns hardcoded in that HTML, we want to see only one dynamic definition for all of them.

2

Also in the Users List, right below the menu, there's an input that says Search. Yup, you guessed it right, you need to add logic to it. What we would like you to do is to filter all the visible data based on whatever you type on that input.

3

And if you haven't had enough fun already, here's another challenge. Do you see those sorting icons on all the table columns? We would like you to add logic to them. Specifically, we would like to see

  • the arrow up when sorting based on that column ascendingly,
  • the arrow down when sorting descendingly,
  • and no icon when we're not sorting based on that column.

4

It's great to see the user's basic info on each table row, but we would like to see their info expanded. That's why we would like you to implement a new component named user-detail.component that renders all the info available for a user (you could check its available properties on the user.interface).

Also, make sure you create a new route where you receive the id of the selected user as a parameter. Oh, and yes, could you please trigger the event to see a user's detail when clicking on a row? Finally, as a side note, remember to add functionality to go back to the table. Maybe a button, whatever floats your boat.

5

We know you might be a bit tired by now, but if not. Here's the last challenge we're sure you're going to love, and who knows, maybe you've done it already. Remember that new component you just created? Well, we would like you to create a reactive form to edit its values in real-time. Make all the fields required, and add a dummy put method to the data.service.

6

Looking for extra points? Awesome! Implement the create mode on the user-detail.component. Meaning that you need to add a route without parameters, since there's no id for new users, and generate a dummy post method in the data.service.

Wow, you've come to the end of this Blitz challenge! But not so fast! Before going away, remember to send the link of your forked solution to your recruiter. She will make sure to propagate your amazing work to us!
Be proud of yourself because we surely are proud of you.
Have a good one.
